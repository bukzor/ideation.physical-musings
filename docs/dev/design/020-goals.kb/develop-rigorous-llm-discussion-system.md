Develop a system to discuss hard topics like G.DIS productively with an
LLM: guard against backslide, sycophancy, and confident subtle errors
(symmetrically, for both parties), and give the user a concrete,
rigorous way to present and evaluate ideas. Design is open
([what-is-the-sys-design]) and derives strictly from settled claims
([technical-policy.kb/design-from-settled]) -- nothing in the current
discussion scaffolding is presumed to be part of the answer.
