# Technical policy

Cross-cutting normative rules for the eventual G.SYS design -- means,
not ends. Each traces `why:` up to the requirement it serves, and its
body should trace down to the settled claim that grounds it
([technical-policy.kb/design-from-settled]). Descriptive rationale
belongs in `../claims.kb/`, not here; this collection states the rule.
